# asty-blog-agent-dashboard

Private control panel for the multi-site blog automation agent.

This is a standalone Next.js app that lives inside the agent repo but
deploys separately. It reads `sites/*/config.json` files from the parent
folder and calls each site's bearer-authenticated admin export endpoints.

## Pages

- `/` — All sites summary (online/offline status, post counts, views)
- `/sites/[id]` — Per-site detail: KPIs, posts table with scores, categories, affiliate readiness
- `/pipeline` — DeepL monthly usage, site inventory, external admin links
- `/reports` — Weekly performance reports generated by `score-posts.ts`

## Local development

```bash
cd dashboard
npm install
cp .env.example .env.local
# Edit .env.local:
#   DASHBOARD_PASSWORD=<set a password>
#   DASHBOARD_SESSION_SECRET=<openssl rand -hex 32>
#   ASTY_AGENT_API_KEY=<from parent .env>
npm run dev
# Visit http://localhost:3000, login with DASHBOARD_PASSWORD
```

## Deployment (Vercel)

Deploy as a **separate Vercel project** from the ASTY Cabin site:

1. Import the `asty-blog-agent` repo, set root directory to `dashboard/`
2. Add environment variables:
   - `DASHBOARD_PASSWORD`
   - `DASHBOARD_SESSION_SECRET`
   - For each site: the env var name referenced by `sites/<id>/config.json` → `env.api_key`
     (e.g. `ASTY_AGENT_API_KEY`, `SITE_B_AGENT_API_KEY`, ...)
3. Include Files: set the `sites/` folder to be included (Vercel respects the repo folder during build)

## Adding a new site to the dashboard

The dashboard auto-discovers any `sites/<id>/config.json`. Once you:
1. Add the config file in the agent repo
2. Set the site's bearer token env var on the dashboard Vercel deployment
3. Redeploy the dashboard

…it shows up automatically on the `/` page.

## Auth

A minimum-viable password login. For stronger auth, wire in NextAuth
with an OAuth provider (Google Workspace, GitHub) later.
