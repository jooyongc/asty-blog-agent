## 2) 운영 원칙

### 2-1. 발행 워크플로 (Phase 9)

```
디렉션 입력  →  3개 주제 제안  →  사용자 1개 👍 선택
            ↓
Director → SEO → Writer → Verifier → Translate → Packager → Linker
            ↓
       승인 대기열에 카드 자동 적재
            ↓
       사용자 Approve 1회 클릭  →  3개 언어 동시 발행 + 그래프 추출
```

**핵심**: 사용자가 직접 하는 일은 "방향 한 문장 쓰기" + "Approve 클릭"뿐. 검수·팩트체크·번역·내부링크는 모두 자동.

### 2-2. Verifier 상태 규칙 (Phase 7-3)

| 상태 | 조건 | 후속 동작 |
|---|---|---|
| `verified` | 모든 주장이 출처로 뒷받침됨 | 바로 대기열로 |
| `partial` | `unsupported ≥ 1` | Writer가 1회 자동으로 완화 표현 재작성 |
| `blocked` | `contradicted ≥ 1` | 대기열 상태 `blocked` → **사람 검토 필수** |
| `skipped` | 검증 단계 생략 옵션 | (기본값은 `verified` 필수) |

### 2-3. 예산 관리 ($5/사이트/월)

대시보드 우측 상단 · 포트폴리오 카드 · 승인 대기열 메트릭에서 항상 `현재 지출 / $5` 노출.

| 프로파일 | 월 예산 범위 | 구성 |
|---|---|---|
| **Lean** (권장) | $4.4~$4.9 | 그래프 추출 격주 · 임베딩 OFF · Director/Verifier 유지 |
| **Standard** | $5.0~$5.3 | 그래프 추출 주 1회 · 임베딩 ON · 피드백 루프 유지 |
| **Full** | $5.5+ | 모든 기능 상시 ON · 예산 초과 허용 사이트에만 |

**자동 제어**:
- $4.5 초과 → 경고 chip 표시
- $5.0 초과 → `budget-guard`가 사이클 자동 중단, 포트폴리오 카드에 "budget-guard가 사이클을 중단함" 배너
- 월말 7일 전 자동 다운시프트 (Full → Lean)

### 2-4. 주간 사이클 (크론: 일요일 21:00 KST)

1. 일요일 21:00 — `/weekly` 커맨드 자동 트리거
2. `topics/manual-queue.md` 상위 3개 주제 선택 (또는 디렉션 페이지 👍 주제)
3. 3편 병렬 파이프라인 실행 (약 3~4분)
4. 승인 대기열에 카드 3개 적재 → **사용자 승인 대기**
5. 사용자가 월요일 오전에 일괄 Approve → 3개 언어 동시 발행

## 3) 에이전트 정의

| 에이전트 | 모델 | 역할 | Phase |
|---|---|---|---|
| `director` | Haiku 4.5 | 자유 텍스트 → 3개 주제 제안, GSC·그래프 맥락 참조 | 9 |
| `seo` | Haiku 4.5 | Striking-distance 키워드 리서치 | 1~6 (10에서 GSC 연동) |
| `writer` | Haiku 4.5 | 리서치 + 초안 작성 + 셀프 편집 | 1~6 |
| `verifier` | Haiku 4.5 | 주장 팩트체크, 모순 감지 시 차단 | **7 (신규)** |
| `translate` | DeepL | EN → JA / ZH-hans, 용어집 적용 | 1~6 |
| `packager` | Haiku 4.5 | 언어별 SEO 메타데이터 + 스키마 (FAQPage, HowTo) | 10에서 확장 |
| `linker` | Graph 기반 | CO_OCCURS 관계로 글당 3~5개 내부링크 자동 삽입 | **10 (신규)** |
| `publish` | HTTP | 사이트 API POST + 그래프 엔티티 추출 트리거 | 1~6 |
